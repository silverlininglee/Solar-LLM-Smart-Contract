# 🛡️ AI 전세 계약 특약사항 진단 시스템 (PoC)

> **Project Overview**: 본 프로젝트는 기존 **사용자 10만 명 규모의 API 기반 리스크 모니터링 서비스** 운영 경험을 바탕으로, **Upstage Solar LLM 및 Document AI**를 활용하여 비정형 데이터(특약사항) 분석의 한계를 극복하기 위해 설계된 기술 고도화 기획안입니다.

---

## 📌 1. Context & Motivation

### 1.1 기존 서비스의 성과와 한계
* **성과**: 등기/시세 API 연동을 통해 리스크 모니터링 구현, 출시 6개월 만에 10만 명의 사용자를 확보하며 전세 안전 서비스의 시장성을 검증함.
* **한계**: 기존 방식은 정형 데이터(등기부등본 등) 분석에 국한되어, 전세 사기의 핵심인 **'비정형 데이터(계약서 특약사항)'** 내 독소 조항 및 세입자 불리 조항을 자동 식별하지 못하는 기술적 블로커가 존재함.

### 1.2 AI를 통한 문제 해결
* 사용자가 직접 읽고 판단해야 했던 복잡한 특약사항을 **Upstage Document Parse(VLM)**와 **Solar LLM**을 결합해 자동 분석함.
* 단순 조회를 넘어 '문맥 해석'을 통한 정성적 위험 판별 영역으로 제품 가치를 확장함.

---

## 🎯 2. 핵심 기능

### 2.1 AI 파이프라인 설계
1. **계약서 촬영/업로드**: 멀티 페이지 및 특약 대응.
2. **Document AI (VLM)**: Upstage Document Parse를 활용해 '특약사항' 영역 탐지 및 텍스트 추출.
3. **Solar LLM 분석**: 8대 위험 카테고리 분류 및 위험도(Level 0~3) 판정.
4. **결과 제공**: 위험 사유 설명 및 구체적인 대응 방안(수정 요청 문구 등) 제안.

---

## ⚙️ 3. 기술 요구사항 (Upstage 제품 활용 방안)

**3.1 Document Parse (계약서를 읽는 똑똑한 '눈')**
* **활용 이유**: 전세 계약서는 형태가 다양하고 표가 섞여 있어 Upstage의 도구를 활용하여 문서의 전체 구조(표, 제목)로 정확하게 글자를 추출합니다.
* **핵심 기능**: 기울어진 사진 보정 및 특약사항 영역만 집중적으로 읽어내는 레이아웃 분석.

**3.2 Solar LLM (내용을 판단하는 법률 '두뇌')**
* **활용 이유**: 읽어온 텍스트가 세입자에게 유리한지 불리한지 문맥을 파악합니다. 한국어와 국내 임대차법에 특화되어 있어 법률 용어의 미묘한 차이를 정확히 해석합니다.
* **핵심 기능**: 분석 결과를 시스템에 바로 활용할 수 있도록 정해진 형식(JSON)으로 답변하는 안정성 확보.

## 📊 4. 모델 평가 지표 (성능 확인 기준)

AI가 얼마나 일을 잘하는지 측정하기 위한 PM의 핵심 기준입니다.

| 지표명 | 목표치 | PM의 관점 (의미) |
| :--- | :--- | :--- |
| **글자 인식률 (CER)** | 97% 이상 | 100글자 중 97글자 이상 정확히 읽기. 분석의 기본 토대입니다. |
| **위험 간과율 (FN)** | **5% 이하** | **[최우선]** 진짜 위험한 조항을 안전하다고 놓칠 확률을 극소화합니다. |
| **과잉 알람률 (FP)** | 15% 이내 | 위험을 놓치는 것보다, 조금이라도 의심되면 확인을 요청하는 보수적 방식을 취합니다. |
| **응답 속도** | 10초 이내 | 사용자가 결과를 기다리다 지치지 않도록 분석 속도를 관리합니다. |

## ❓ 5. 주요 의사결정 및 트레이드오프 (Key Decisions)

1. **[정확도 우선 정책]**: 운영 비용을 아끼기 위해 저사양 모델을 쓰기보다, 사용자의 보증금을 보호하기 위해 최고 성능의 **Solar Pro 모델**을 선택했습니다.
2. **[보수적 탐지 전략]**: 위험 조항을 놓치는 사고를 막기 위해, 판단이 모호한 조항은 '주의' 단계로 분류하여 사용자에게 알리는 정책을 수립했습니다.
3. **[보안 최우선]**: 민감한 개인정보인 계약서 이미지는 분석 완료 즉시 **서버에서 삭제**하는 휘발성 아키텍처를 결정했습니다.

## 🧪 Model Optimization & Evaluation (실험 및 개선)

단순 기획을 넘어, Upstage Solar 모델의 성능을 최적화하기 위한 3단계 실험을 진행했습니다.



### Phase 1: Zero-shot Baseline
- **방법**: 별도 가이드 없이 특약사항 위험도 문의
- **한계**: 답변의 형식이 불분명하고 법률적 근거 부족

### Phase 2: Role-Play & Task Structuring
- **방법**: '임대차보호법 전문가' 페르소나 부여 및 8대 위험 카테고리(보증금, 근저당 등) 정의
- **성과**: 카테고리별 분류 정확도 상승

### Phase 3: Few-shot Prompting (최종 채택)
- **방법**: 실제 판례 기반의 위험/안전 사례를 예시 데이터로 제공
- **결과**: 위험도 판별 일관성 30% 향상, 특히 '보증금 반환 유예' 조항에 대한 탐지 정밀도 확보

- VLM(Document Parse)으로 읽어온 비정형 데이터의 노이즈를 LLM 단계에서 어떻게 정제하느냐가 제품 신뢰도의 핵심임을 파악함.
